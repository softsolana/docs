---
title: "Publish option"
description: "Create a new prediction market with named outcomes and a fee rate."
---

`publish_option` creates a `Market` account that defines a prediction market. Once published, users can trade outcome shares on it via [`swap`](/contracts/predict/swap).

Outcome `0` — **Inconclusive** — is always prepended by the contract. The outcomes you supply start at index `1`.

## Parameters

<ParamField body="option_identifier" type="string" required>
  Unique identifier for this market scoped to the operator. Max 50 characters. Used to derive the market PDA.
</ParamField>

<ParamField body="ancillary_data" type="string" required>
  Human-readable description or metadata for the market — the question being predicted, resolution criteria, data sources, etc. Max 200 characters.
</ParamField>

<ParamField body="fees" type="u64" required>
  Operator fee in basis points, charged on each swap. A platform fee of 25 bps is applied on top.
</ParamField>

<ParamField body="outcomes" type="string[]" required>
  Named outcomes for the market. Must contain between 2 and 99 entries. Do not include "Inconclusive" — it is prepended automatically as index 0.
</ParamField>

## Accounts

| Account | Writable | Description |
|---|:---:|---|
| `option` | ✓ | Market PDA. Created by this instruction. |
| `operator` | ✓ | Operator wallet. Signs the transaction and pays rent for the market PDA. |
| `system_program` | | |

## Errors

| Code | Error | Cause |
|---|---|---|
| `6013` | `InvalidOutcomes` | Fewer than 2 outcomes provided. |
| `6025` | `TooManyOutcomes` | More than 99 outcomes provided. |
