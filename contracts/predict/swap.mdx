---
title: "Swap"
description: "Buy outcome shares with USDC or sell shares back for USDC."
---

`swap` lets a user buy or sell shares in a specific outcome on an open outcome market. Prices are set by the LMSR pricing model and update with each trade. If the outcome a user holds wins, their shares redeem 1:1 for USDC.

Use [`get_quote`](/contracts/predict/get-quote) first to preview the trade and determine a safe `slippage_limit`.

## Parameters

<ParamField body="option_identifier" type="string" required>
  The market identifier used when the market was published.
</ParamField>

<ParamField body="outcome_index" type="u8" required>
  Index of the outcome to buy or sell. Must be `1` or higher — outcome `0` (Inconclusive) cannot be traded.
</ParamField>

<ParamField body="is_buy" type="bool" required>
  - `true` — spend USDC to buy outcome shares. `amount` is the total USDC including fees.
  - `false` — sell outcome shares for USDC. `amount` is the number of shares to sell.
</ParamField>

<ParamField body="amount" type="u64" required>
  USDC to spend in base units when buying, or number of shares when selling. Must be greater than zero.
</ParamField>

<ParamField body="slippage_limit" type="u64" required>
  Minimum acceptable output. For buys, the minimum shares to receive. For sells, the minimum USDC to receive. The transaction fails with `SlippageExceeded` if the actual output falls below this value.
</ParamField>

## Errors

| Code | Error | Cause |
|---|---|---|
| `6026` | `TradingInactive` | The market has already been resolved. |
| `6012` | `InvalidOutcome` | `outcome_index` is out of range or is `0` (Inconclusive). |
| `6010` | `InvalidAmount` | `amount` is zero. |
| `6008` | `InsufficientTokens` | Selling more shares than the user holds. |
| `6024` | `SlippageExceeded` | Output fell below `slippage_limit`. |
| `6017` | `MathOverflow` | Arithmetic overflow in share or payout calculation. |
