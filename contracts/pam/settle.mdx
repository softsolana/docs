---
title: "Settle"
description: "Evaluate a bet's payout matrix and pay out the result."
---

`settle` reads `resolved_slots` from the linked `Resolution` account, evaluates the bet's payout matrix row by row, and transfers the appropriate amount to the user. The bet and escrow accounts are closed at the end of settlement.

Anyone can call `settle` â€” no specific authority is required. The Resolution must be `Finalized` before settlement can proceed.

## Parameters

<ParamField body="id" type="u64" required>
  The same `id` used when the bet was placed. Used to derive the bet and escrow PDAs.
</ParamField>

## Payout evaluation

Rows in the payout matrix are tested top-to-bottom against the resolution's `resolved_slots`:

- The **first row whose conditions all pass** determines the payout.
- A `null` payout_amount is a **void/push** â€” the full stake is returned to the user.
- If **no row matches**, the bet is a **loss** â€” the stake goes to the treasury.

The payout is capped at the escrow balance. Any remainder after the payout is returned to the active treasury, and the escrow account is closed.

## Accounts

| Account | Writable | Description |
|---|:---:|---|
| `settler` | | Transaction signer. Can be anyone. |
| `platform` | | Platform wallet. Used to derive PDAs. |
| `user` | âś“ | The original bet placer. Receives payout and reclaimed rent. |
| `bet_escrow` | âś“ | Escrow PDA. Closed at settlement. |
| `bet` | âś“ | Bet descriptor PDA. Closed at settlement; rent returned to user. |
| `resolution` | âś“ | The Resolution linked to this bet. Must be Finalized. |
| `usdc_mint` | | USDC mint. |
| `user_account` | âś“ | User's USDC associated token account. Receives the payout. |
| `active_treasury` | âś“ | Platform treasury. Receives any unspent escrow funds. |
| `system_program` | | |
| `token_program` | | |
| `associated_token_program` | | |

## Errors

| Error | Cause |
|---|---|
| `Unauthorized` | The linked Resolution is not yet Finalized. |
| `MathOverflow` | Arithmetic overflow computing escrow remainder. |

## Example

```typescript
import * as anchor from "@coral-xyz/anchor";
import { getAssociatedTokenAddress } from "@solana/spl-token";

const USDC_MINT = new anchor.web3.PublicKey("EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v");

const id = new anchor.BN(1);
const idBytes = id.toArrayLike(Buffer, "le", 8);

const [betEscrow] = anchor.web3.PublicKey.findProgramAddressSync(
  [Buffer.from("ss_escrow"), platform.publicKey.toBuffer(), user.publicKey.toBuffer(), idBytes],
  program.programId
);
const [bet] = anchor.web3.PublicKey.findProgramAddressSync(
  [Buffer.from("ss_bet"), platform.publicKey.toBuffer(), user.publicKey.toBuffer(), idBytes],
  program.programId
);
const [resolution] = anchor.web3.PublicKey.findProgramAddressSync(
  [Buffer.from("ss_resolution"), platform.publicKey.toBuffer(), idBytes],
  program.programId
);
const [activeTreasury] = anchor.web3.PublicKey.findProgramAddressSync(
  [Buffer.from("ss_active_tres"), platform.publicKey.toBuffer()],
  program.programId
);

const userAccount = await getAssociatedTokenAddress(USDC_MINT, user.publicKey);

await program.methods
  .settle(id)
  .accounts({
    settler: settler.publicKey,
    platform: platform.publicKey,
    user: user.publicKey,
    betEscrow,
    bet,
    resolution,
    usdcMint: USDC_MINT,
    userAccount,
    activeTreasury,
  })
  .signers([settler])
  .rpc();
```
