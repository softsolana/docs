---
title: "Place"
description: "Create a new bet and lock collateral into escrow."
---

`place` creates a `Bet` account linked to a `Resolution` PDA, transfers the user's stake into escrow, and reserves the maximum possible payout from the active treasury alongside it. This guarantees funds are available at settlement before the outcome is known.

Calling `place` again with the same `id` **adjusts** an existing bet — the stake and payout matrix can both be changed. The contract settles any difference with the user and treasury automatically. Adjustments are only possible while the linked Resolution is still `Pending`.

A protocol fee is deducted from the stake at placement and routed to the yield vault for stakers.

## Parameters

<ParamField body="stake" type="u64" required>
  Bet size in USDC base units (1 USDC = 1,000,000). Transferred in full from the user at placement.
</ParamField>

<ParamField body="id" type="u64" required>
  Unique identifier for this bet, scoped to the (platform, user) pair. Used to derive the bet and escrow PDAs. Must not be reused while a bet with that `id` is still open.
</ParamField>

<ParamField body="payout_matrix" type="PayoutRow[]" required>
  Ordered list of payout rules evaluated at settlement. See [payout matrix](#payout-matrix) below.
</ParamField>

## Payout matrix

A `PayoutRow` describes a win condition and its absolute payout amount. Rows are evaluated top-to-bottom at settlement; the first matching row determines the payout. If no row matches, the bet is a loss.

```typescript
type PayoutRow = {
  conditions: Condition[];       // all must pass; empty array = always matches
  payout_amount: number | null;  // absolute USDC base units; null = void/push (return stake)
};

type Condition = {
  o1: number;           // index into resolved_slots (left-hand side)
  o2: number | null;    // if set, rhs = resolved_slots[o2] (slot vs slot comparison)
  op: "Gt" | "Gte" | "Lt" | "Lte" | "Eq" | "Neq";
  value: number;        // literal rhs, scaled ×1,000,000; ignored when o2 is set
};
```

The escrow is funded with `stake + max(payout_amount)` across all rows. The active treasury must hold enough to cover the maximum payout at the time of placement.

<Note>
All `resolved_slots` values and literal `value` fields in conditions are scaled by 1,000,000. A score of 3 is represented as `3_000_000`.
</Note>

### Example: three-way soccer match (1X2)

Each selection is a separate bet. The `resolved_slots` for this resolution are `[home_score, away_score]`.

```typescript
// User bets 10 USDC on home win — pays out 25 USDC if home score > away score
const homeWinMatrix = [
  {
    conditions: [{ o1: 0, o2: 1, op: "Gt", value: 0 }],
    payout_amount: 25_000_000,
  },
];

// User bets 10 USDC on draw — pays out 30 USDC if scores are equal
const drawMatrix = [
  {
    conditions: [{ o1: 0, o2: 1, op: "Eq", value: 0 }],
    payout_amount: 30_000_000,
  },
];

// User bets 10 USDC on away win — pays out 27 USDC if away score > home score
const awayWinMatrix = [
  {
    conditions: [{ o1: 1, o2: 0, op: "Gt", value: 0 }],
    payout_amount: 27_000_000,
  },
];
```

<Note>
Slot-vs-slot comparisons (e.g. `home_score > away_score`) use `o2`. Set `value` to `0` — it is ignored when `o2` is provided.

Literal comparisons (e.g. home team scored more than 2 goals) use `value` scaled by 1,000,000: `{ o1: 0, o2: null, op: "Gt", value: 2_000_000 }`.
</Note>

## Accounts

| Account | Writable | Description |
|---|:---:|---|
| `operator` | | Authorised operator key. Determines the protocol fee rate. |
| `platform` | | Platform wallet. Used to derive the active treasury and bet PDAs. |
| `user` | ✓ | Bet placer. Signs the transaction, pays the stake and rent for new PDAs. |
| `bet_escrow` | ✓ | PDA token account. Holds `stake + max_win` until settlement. |
| `bet` | ✓ | PDA storing the bet descriptor. Closed at settlement. |
| `resolution` | ✓ | The Resolution this bet will settle against. Must not be Finalized. |
| `usdc_mint` | | USDC mint. |
| `user_account` | ✓ | User's USDC associated token account. Source of the stake. |
| `active_treasury` | ✓ | Platform treasury PDA. Source of the max payout collateral. |
| `yield_vault` | ✓ | Protocol vault. Receives the protocol fee. |
| `yield_state` | ✓ | Global staking state. Updated when fee is distributed. |
| `system_program` | | |
| `token_program` | | |
| `associated_token_program` | | |

## Errors

| Error | Cause |
|---|---|
| `InvalidAmount` | `stake` is zero (first placement). |
| `EmptyPayoutMatrix` | `payout_matrix` is empty (first placement). |
| `Unauthorized` | The linked Resolution is already Finalized. |
| `MathOverflow` | Arithmetic overflow in fee or escrow calculation. |

## Example

```typescript
import * as anchor from "@coral-xyz/anchor";

const OPERATOR = new anchor.web3.PublicKey("cxa3sYKNMk5L8PvDqEzWWLAfd4hr88d6U25W6Yz3FiE");
const id = new anchor.BN(1);

// Bet 10 USDC on home win — pays out 25 USDC if home score > away score
const payoutMatrix = [
  {
    conditions: [{ o1: 0, o2: 1, op: { gt: {} }, value: new anchor.BN(0) }],
    payoutAmount: new anchor.BN(25_000_000),
  },
];

await program.methods
  .place(new anchor.BN(10_000_000), id, payoutMatrix)
  .accounts({
    operator: OPERATOR,
    platform: platform.publicKey,
    user: user.publicKey,
    resolution, // the Resolution PDA created by request_resolution
  })
  .signers([user])
  .rpc();
```
