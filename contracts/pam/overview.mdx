The Player Account Management (PAM) program handles the full lifecycle of a bet — from placement through to settlement.

## Complete flow

```mermaid
flowchart LR
    PL[Platform] -->|request_resolution| RES[Resolution Pending]

    US[User] -->|place| ESC[Bet Escrow (stake + max_win)]
    ESC --> RES

    PL -->|submit_resolution| FIN[Resolution Finalized]
    RES --> FIN

    ANY[Anyone] -->|settle| PAY{Payout Matrix}
    FIN --> PAY

    PAY -- Win --> UW[Payout to User]
    PAY -- Loss --> TR[Return to Treasury]
    PAY -- Void/Push --> RS[Stake Returned]

    UW --> CL[Close Escrow]
    TR --> CL
    RS --> CL
```

## Key concepts

<AccordionGroup>
  <Accordion title="Resolution account">
    Every bet is linked to a **Resolution** PDA created by the platform via `request_resolution`. The resolution defines how the outcome will be determined (single authority, provably fair randomness, or consensus) and holds the final `resolved_slots` values once submitted. See the [Resolution](/contracts/resolution/overview) section for details on each method.
  </Accordion>
  <Accordion title="Payout matrix">
    Each bet defines an ordered list of rows. Each row has one or more conditions on `resolved_slots` values and an absolute `payout_amount` in USDC base units. Rows are evaluated top-to-bottom; the first matching row wins. A `null` payout_amount is a void/push and returns the stake. If no row matches, the bet is a loss.
  </Accordion>
  <Accordion title="Escrow">
    At placement, USDC is locked into a per-bet escrow PDA holding `stake + max_win`. The `max_win` is sourced from the active treasury. This guarantees the maximum payout is available before settlement. The escrow is closed at the end of settlement and any remainder is returned to the treasury.
  </Accordion>
  <Accordion title="Adjustable bets">
    A bet can be modified before its Resolution finalizes by calling `place` again with the same `id`. The stake and payout matrix can both be updated — the contract reconciles the difference with the user and treasury automatically.
  </Accordion>
  <Accordion title="Protocol fee">
    A fee in basis points is deducted from each stake at placement and routed to the yield vault for stakers. The default rate for third-party platforms is 10 bps. SoftSol-native contracts receive a lower rate. The fee does not reduce escrow collateral — `stake + max_win` is always fully funded.
  </Accordion>
</AccordionGroup>
