---
title: "Request resolution"
description: "Create a Resolution account and choose how its outcome will be determined."
---

`request_resolution` creates a `Resolution` PDA with status `Pending` and binds it to a resolution method. Bets are linked to this account at placement and settle against it once it is finalized.

The platform signs this instruction and pays rent for the new account.

## Parameters

<ParamField body="id" type="u64" required>
  Unique identifier for this resolution, scoped to the platform. Used to derive the Resolution PDA. Must not be reused while a resolution with that `id` is still open.
</ParamField>

<ParamField body="method" type="ResolutionMethod" required>
  Determines who can submit the outcome and how it is verified. See [resolution methods](#resolution-methods) below.
</ParamField>

<ParamField body="ancillary" type="bytes" required>
  UTF-8 encoded metadata for this resolution â€” up to 512 bytes. Must contain at least a question line and an outcomes line:

  ```
  q: Will team A beat team B?
  o: yes, no
  ```

  - `q:` â€” the question being resolved (required, must be non-empty)
  - `o:` â€” comma-separated list of possible outcomes (required, at least one non-empty value)

  Additional lines are permitted and can carry any extra context.
</ParamField>

## Resolution methods

<Warning>
  The Consensus resolution method is currently disabled.
</Warning>

<AccordionGroup>
  <Accordion title="Single Source">
    A designated authority submits the outcome directly. An optional co-authority can be required for 2-of-2 sign-off, and an optional deadline enforces submission before a given Unix timestamp.

    ```typescript
    {
      SingleSource: {
        authority: PublicKey,        // required â€” must sign submit_resolution
        deadline: number | null,     // optional Unix timestamp; submission rejected after this
        co_authority: PublicKey | null, // optional â€” must also sign submit_resolution
      }
    }
    ```
  </Accordion>
  <Accordion title="ProvablyFair">
    The platform commits to a `seed_hash` (SHA-256 of a secret server seed) upfront. When submitting, the server seed is revealed and the outcome is derived deterministically â€” verifiable by anyone.

    ```typescript
    {
      ProvablyFair: {
        seed_hash: Uint8Array,      // 32-byte SHA-256 of the server seed; must be non-zero
        result: null,               // always null at request time; filled in by submit_resolution
        selection_mode: SelectionMode,
      }
    }
    ```

  </Accordion>
  <Accordion title="Consensus">
    Disabled
  </Accordion>
</AccordionGroup>

## Errors

| Error | Cause |
|---|---|
| `AncillaryTooLarge` | `ancillary` exceeds 512 bytes. |
| `AncillaryNotUtf8` | `ancillary` is not valid UTF-8. |
| `AncillaryMissingQuestion` | `ancillary` has no `q:` line, or the question is empty. |
| `AncillaryMissingOutcomes` | `ancillary` has no `o:` line, or all listed outcomes are empty. |
| `ConsensusTempDisabled` | `Consensus` method was passed â€” currently disabled. |
| `InvalidSeedHash` | `ProvablyFair` seed_hash is all zeros. |

## Example

<CodeGroup>
```typescript SingleSource
import * as anchor from "@coral-xyz/anchor";

const id = new anchor.BN(1);

const [resolution] = anchor.web3.PublicKey.findProgramAddressSync(
  [Buffer.from("ss_resolution"), platform.publicKey.toBuffer(), id.toArrayLike(Buffer, "le", 8)],
  program.programId
);

const ancillary = Buffer.from(
  "q: Will team A beat team B?\no: yes, no",
  "utf-8"
);

await program.methods
  .requestResolution(id, { singleSource: { authority: authority.publicKey, deadline: null, coAuthority: null } }, ancillary)
  .accounts({
    platform: platform.publicKey,
    resolution,
  })
  .signers([platform])
  .rpc();
```

```typescript ProvablyFair
import * as anchor from "@coral-xyz/anchor";
import { sha256 } from "@noble/hashes/sha256";

const id = new anchor.BN(2);

const [resolution] = anchor.web3.PublicKey.findProgramAddressSync(
  [Buffer.from("ss_resolution"), platform.publicKey.toBuffer(), id.toArrayLike(Buffer, "le", 8)],
  program.programId
);

const serverSeed = crypto.getRandomValues(new Uint8Array(32));
const seedHash = sha256(serverSeed); // commit â€” keep serverSeed secret until submit

const ancillary = Buffer.from(
  "q: Which side will win the coin flip?\no: heads, tails",
  "utf-8"
);

await program.methods
  .requestResolution(
    id,
    {
      provablyFair: {
        seedHash: Array.from(seedHash),
        result: null,
        selectionMode: { simple: { numSlots: 2 } },
      },
    },
    ancillary
  )
  .accounts({
    platform: platform.publicKey,
    resolution,
  })
  .signers([platform])
  .rpc();
```
