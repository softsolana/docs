---
title: "Request resolution"
description: "Create a Resolution account and choose how its outcome will be determined."
---

`request_resolution` creates a `Resolution` PDA with status `Pending` and binds it to a resolution method. Bets are linked to this account at placement and settle against it once it is finalized.

The platform signs this instruction and pays rent for the new account.

## Parameters

<ParamField body="id" type="u64" required>
  Unique identifier for this resolution, scoped to the platform. Used to derive the Resolution PDA. Must not be reused while a resolution with that `id` is still open.
</ParamField>

<ParamField body="method" type="ResolutionMethod" required>
  Determines who can submit the outcome and how it is verified. See [resolution methods](#resolution-methods) below.
</ParamField>

<ParamField body="ancillary" type="bytes" required>
  Arbitrary off-chain metadata — up to 512 bytes. Use this to store a reference to the event, market identifier, or any other context needed by resolvers or clients. Pass an empty array if not needed.
</ParamField>

## Resolution methods

<AccordionGroup>
  <Accordion title="SingleSource">
    A designated authority submits the outcome directly. An optional co-authority can be required for 2-of-2 sign-off, and an optional deadline enforces submission before a given Unix timestamp.

    ```typescript
    {
      SingleSource: {
        authority: PublicKey,        // required — must sign submit_resolution
        deadline: number | null,     // optional Unix timestamp; submission rejected after this
        co_authority: PublicKey | null, // optional — must also sign submit_resolution
      }
    }
    ```

    See [Single Source](/contracts/resolution/single-source) for more detail.
  </Accordion>
  <Accordion title="ProvablyFair">
    The platform commits to a `seed_hash` (SHA-256 of a secret server seed) upfront. When submitting, the server seed is revealed and the outcome is derived deterministically — verifiable by anyone.

    ```typescript
    {
      ProvablyFair: {
        seed_hash: Uint8Array,      // 32-byte SHA-256 of the server seed; must be non-zero
        result: null,               // always null at request time; filled in by submit_resolution
        selection_mode: SelectionMode,
      }
    }
    ```

    See [Provably Fair](/contracts/resolution/provably-fair) for details on `SelectionMode` and outcome derivation.
  </Accordion>
  <Accordion title="Consensus (disabled)">
    Stake-weighted voting. Currently disabled — `request_resolution` will reject any `Consensus` method with a `ConsensusTempDisabled` error.

    See [Consensus](/contracts/resolution/consensus) for how this method will work when enabled.
  </Accordion>
</AccordionGroup>

## Accounts

| Account | Writable | Description |
|---|:---:|---|
| `platform` | ✓ | Platform wallet. Signs the transaction and pays rent for the Resolution PDA. |
| `resolution` | ✓ | Resolution PDA. Created by this instruction. |
| `system_program` | | |

## Errors

| Error | Cause |
|---|---|
| `AncillaryTooLarge` | `ancillary` exceeds 512 bytes. |
| `ConsensusTempDisabled` | `Consensus` method was passed — currently disabled. |
| `InvalidSeedHash` | `ProvablyFair` seed_hash is all zeros. |
