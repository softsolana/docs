---
title: "Provably fair"
description: "Resolve outcomes using a verifiable commit-reveal scheme."
---

The `ProvablyFair` method produces outcomes that are cryptographically verifiable. The platform commits to a secret server seed before any bets are placed. The outcome is derived deterministically from that seed at resolution time — neither the platform nor the user can manipulate it after commitment.

## How it works

1. Before `request_resolution`, the platform generates a secret `server_seed` and computes `seed_hash = SHA-256(server_seed)`.
2. At `request_resolution`, `seed_hash` and a `selection_mode` are committed on-chain. The server seed remains secret.
3. Bets are placed against the `Pending` resolution.
4. At `submit_resolution`, the platform reveals `server_seed`. The contract verifies `SHA-256(server_seed) == seed_hash`, then derives the result:
   ```
   result = SHA-256(server_seed || client_seed)
   ```
5. `resolved_slots` is computed from `result` according to the `selection_mode`. Status moves to `Finalized`.

Anyone can verify the outcome independently using the revealed `server_seed` and the `client_seed`.

## Client seed

An optional `client_seed` can be mixed into the result hash. This allows users or the platform to contribute entropy, further reducing the trust requirement. If no `client_seed` is provided, an empty byte array is used.

## Selection modes

The `selection_mode` determines how the 32-byte `result` hash is mapped to `resolved_slots`.

<AccordionGroup>
  <Accordion title="Simple">
    One winner is selected from `num_slots` outcomes with equal probability. Produces a one-hot output where the winning index is `1` and all others are `0`.

    ```
    winner = result[0] % num_slots
    resolved_slots = [0, 0, 1, 0]  // example with num_slots = 4, winner = 2
    ```

    Use for coin flips, binary outcomes, or any equal-probability selection.
  </Accordion>
  <Accordion title="Weighted">
    One winner is selected via a cumulative distribution over a `weights` array. Produces a one-hot output. Outcomes with higher weights are more likely to be selected.

    ```typescript
    // weights: [50, 30, 20] → 50%, 30%, 20% probability
    // pick = u16(result[0..2]) % sum(weights)
    // winner = first index where cumulative weight > pick
    resolved_slots = [0, 1, 0]  // example: index 1 won
    ```

    Use for weighted random outcomes — loot tables, variable-odds games, etc.
  </Accordion>
  <Accordion title="Modulo">
    Produces multiple independent random values. Each slot is derived from a different byte of `result`:

    ```
    resolved_slots[i] = (result[i % 32] % divisor) × 1_000_000
    ```

    Use for card draws, dice rolls, or any game requiring multiple independent random outputs.

    ```typescript
    // Example: 2 dice (divisor = 6)
    // result[0] = 0x3A (58) → 58 % 6 = 4 → 4_000_000
    // result[1] = 0x11 (17) → 17 % 6 = 5 → 5_000_000
    resolved_slots = [4_000_000, 5_000_000]
    ```
  </Accordion>
</AccordionGroup>

## Configuration

```typescript
{
  ProvablyFair: {
    seed_hash: Uint8Array,       // 32-byte SHA-256 of server_seed; must be non-zero
    result: null,                // filled in by submit_resolution
    selection_mode: SelectionMode,
  }
}
```
