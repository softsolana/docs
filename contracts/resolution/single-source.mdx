---
title: "Single source"
description: "Resolve outcomes using a single trusted authority."
---

The `SingleSource` method lets a designated authority write the outcome of a resolution directly. It is the simplest method and suits any bet whose outcome is observed off-chain — sports results, prediction markets, price triggers, and so on.

## How it works

1. At `request_resolution`, the platform specifies an `authority` public key. Optionally, a `co_authority` and a `deadline` can also be set.
2. The resolution remains `Pending` until `submit_resolution` is called.
3. At `submit_resolution`, the authority signs the transaction and provides the `resolved_slots` values. The contract writes them directly — no further verification is performed.
4. Status moves to `Finalized` and all linked bets can be settled.

## Configuration

```typescript
{
  SingleSource: {
    authority: PublicKey,           // must sign submit_resolution
    deadline: number | null,        // optional Unix timestamp; submission fails after this
    co_authority: PublicKey | null, // optional second signer for 2-of-2 sign-off
  }
}
```

### Deadline

If `deadline` is set, `submit_resolution` will be rejected once `clock.unix_timestamp > deadline`. Use this to enforce that outcomes cannot be submitted after a reasonable window has passed.

### Co-authority (2-of-2)

If `co_authority` is set, both `authority` and `co_authority` must sign the same `submit_resolution` transaction. This is useful when two independent parties — for example, an operator and an auditor — must agree on the outcome before it is accepted.

## Resolved slots

With `SingleSource`, `resolved_slots` is whatever the authority provides. The meaning of each slot is defined by how the platform constructs the payout matrix at bet placement.

Common patterns:

| Use case | resolved_slots example |
|---|---|
| Two-team match (home score, away score) | `[3_000_000, 1_000_000]` |
| Binary outcome (win = 1, loss = 0) | `[1_000_000]` |
| Price trigger (price × 1,000,000) | `[42_500_000_000]` |
