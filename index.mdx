## Key concepts

**Betting** — users place bets by defining outcome slots, a payout matrix, and a set of resolvers. Resolvers attest results off-chain and submit Ed25519 signatures at settlement. Random-outcome bets use Pyth Entropy for provable on-chain randomness.

**Staking** — users deposit tokens to earn a pro-rata share of the protocol fee generated by every bet. Rewards accumulate continuously and can be claimed at any time.

**Resolvers** — off-chain services that observe real-world outcomes and sign attestations. Each bet defines its own resolver set and signature threshold, enabling flexible trust models from a single trusted operator to fully decentralised M-of-N schemes.
